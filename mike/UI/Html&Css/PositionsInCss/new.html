1. Static (Default Position)

Every HTML element is positioned static by default. It means the element follows the normal document flow and cannot be moved using top, left, etc.

I am static. I sit where I'm told in the flow.
position: static;

2. Relative Position

An element with position: relative is moved relative to its original position. The space it occupied in the document flow is preserved.

I am relatively shifted right & down

position: relative;
top: 20px;
left: 30px;

3. Absolute Position

With position: absolute, the element is removed from the document flow and is positioned relative to the nearest ancestor with a position other than static. If none, it will use the <html> (viewport) as reference.

I am absolutely positioned in parent

.parent {
  position: relative;
}
.child {
  position: absolute;
  top: 20px;
  left: 30px;
}

4. Fixed Position

A fixed element is removed from normal flow and always stays in the same place relative to the viewport, even while scrolling. Common use: sticky footer, fixed navigation, alerts.

I'm always in bottom-right!

position: fixed;
bottom: 10px;
right: 10px;

5. Sticky Position

A sticky element behaves like relative until it crosses a scroll threshold, then it behaves like fixed. It's useful for sticky headers.

I stick to top when scrolling starts!
Scroll this section to see the sticky behavior. 

position: sticky;
top: 0;

6. Styling Unordered Lists with CSS

HTML doesn't provide much styling control on lists by default. Using CSS, you can modify list-style-type, use images, and even adjust spacing.

  • Custom styled bullet
  • Using square bullets
  • Better indentation

ul.custom-list {
  list-style-type: square;
  padding-left: 25px;
  color: #34495e;
}
